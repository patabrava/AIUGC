# Reference-Image Video Generation Handoff

## Context

This worktree implements and verifies the Veo reference-image path for AIUGC video generation.

Scope:
- Base 8s video generation with three reference images.
- Duration-routed extension chains for 16s and 32s videos.
- Live verification against the real dev server, Supabase, Google Veo, and Cloudflare R2.

The goal was not only to make the requests shape-correct, but to prove the whole flow end-to-end on real provider surfaces.

## What Changed

### Reference images for base generation

The Veo adapter now sends `referenceImages` as `asset` references for base video generation when reference images are enabled.

Relevant code:
- [app/adapters/veo_client.py](/Users/camiloecheverri/Documents/AI/AIUGC/AIUGC/.worktrees/reference-image-video-generation-impl/app/adapters/veo_client.py)
- [app/features/videos/handlers.py](/Users/camiloecheverri/Documents/AI/AIUGC/AIUGC/.worktrees/reference-image-video-generation-impl/app/features/videos/handlers.py)

Behavior:
- Up to three local reference PNGs are loaded.
- The base submit path omits `negativePrompt` when reference images are present.
- The base request uses the current preview model `veo-3.1-generate-preview`.

### Duration routing

The app routes:
- `8s` as a single-shot base video.
- `16s` as an extended chain with one continuation hop.
- `32s` as an extended chain with three continuation hops.

Relevant code:
- [app/core/video_profiles.py](/Users/camiloecheverri/Documents/AI/AIUGC/AIUGC/.worktrees/reference-image-video-generation-impl/app/core/video_profiles.py)

- [workers/video_poller.py](/Users/camiloecheverri/Documents/AI/AIUGC/AIUGC/.worktrees/reference-image-video-generation-impl/workers/video_poller.py)

### Extension request fix

I found a real provider mismatch during live testing:

- The extension submit path was sending `durationSeconds: 7`.
- Google’s extension flow rejects that field for this contract.
- The fix removes `durationSeconds` from the extension payload and keeps the hop length implicit.

Relevant code:
- [app/adapters/veo_client.py](/Users/camiloecheverri/Documents/AI/AIUGC/AIUGC/.worktrees/reference-image-video-generation-impl/app/adapters/veo_client.py)
